## 🎯 الهدف العام

وضع إطار زمني وخطة تنفيذ واضحة ومنهجية مع تقسيم العمل إلى **8 مراحل رئيسية**، بما يضمن تحقيق جميع المتطلبات بدون إسقاط أي ميزة، مع مرونة كاملة للمطورين لإعادة هيكلة الواجهات والأنظمة الداخلية حسب أفضل معايير UX الحديثة.

## 📋 التقسيم الرئيسي للمراحل

### 🔴 المرحلة 1️⃣: تحسينات تجربة المستخدم والواجهات (UI/UX Global Enhancements)

**الأولوية:** عالية جداً | **المدة المتوقعة:** 3-4 أسابيع

#### 1.1 إعادة تصميم الشاشات الداخلية
- [ ] إعادة تصميم كاملة لجميع الصفحات بعد تسجيل الدخول (Internal Dashboard)
  - [ ] لوحة تحكم المتدرب (Student Dashboard)
  - [ ] لوحة تحكم الأدمن (Admin Panel)
  - [ ] لوحة تحكم المدرب (Instructor Dashboard)
- [ ] تطبيق مظهر احترافي ومميز (Premium Look & Feel)
- [ ] اتساق الألوان والخطوط والمباعدات (Design System)

#### 1.2 إصلاح وتحسين النوافذ المنبثقة (Popups)
- [ ] حل مشكلة ظهور نصف نافذة الـ Popup للمتدرب
- [ ] ضبط الـ Responsiveness كاملاً على جميع الأجهزة
- [ ] تحسين المظهر الجمالي والهيكلي لجميع الـ Popups
- [ ] ضمان اتساق Popups في صفحات المتدرب والأدمن

#### 1.3 تحسين شعار المنصة (Logo)
- [ ] زيادة حجم اللوجو بشكل متناسق واحترافي
- [ ] تطبيق التحسين في صفحات تسجيل دخول الأدمن والمتدربين
- [ ] ضمان جودة اللوجو على جميع أحجام الشاشات

#### 1.4 دعم اللغتين (Localization - i18n)
- [ ] تفعيل الخطوط العربية والإنجليزية المتوافقة
- [ ] تطبيق دعم الاتجاهات (RTL/LTR)
- [ ] ضبط النظام لعرض المحتوى بناءً على لغة تصفح المستخدم (Auto-detection)
- [ ] تطبيق على جميع أنحاء الموقع ولوحة التحكم

#### 1.5 تحديثات لحظية بدون تحديث الصفحة (Real-time Updates)
- [ ] ربط لوحة التحكم بالموقع والـ Dashboard بتقنية Sockets
- [ ] تطبيق State Management موحد (Redux/Zustand/Context)
- [ ] ضمان ظهور التعديلات فوراً دون الحاجة لإعادة تحميل الصفحة
- [ ] إلغاء عمل الـ Refresh التلقائي المتكرر في قسم الكورسات للمتدرب

#### 1.6 تحسين الأداء والسرعة (Performance Optimization)
- [ ] فحص شامل للأكواد البرمجية
- [ ] حذف المكتبات الزائدة والموارد غير المستخدمة
- [ ] تنظيف وتحسين الأكواد (Code Clean-up)
- [ ] ضمان أن يكون الموقع خفيفاً وسريعاً وسلس التصفح
- [ ] اختبار الأداء على الهواتف وأجهزة الكمبيوتر

#### 1.7 إعادة تصميم أيقونة الدعم الفني
- [ ] تصميم حديث واحترافي
- [ ] ضمان سهولة الوصول والاستخدام

### 🟠 المرحلة 2️⃣: تحديثات الموقع الرئيسي (Main Website Controls via Admin)

**الأولوية:** عالية | **المدة المتوقعة:** 3-4 أسابيع

#### 2.1 تعديل نصوص المسارات والبرامج
- [ ] تغيير جملة "إدارة التدريب والتطوير" إلى "برامج التدريب والتطوير" (أسفل قسم البرامج)
- [ ] تغيير النص في المسار الأول من "برامج التدريب والتطوير" إلى "إدارة التدريب والتطوير"

#### 2.2 إعادة هيكلة قسم NEW
- [ ] إعادة ترتيب وتصميم قسم NEW في الموقع الرئيسي
- [ ] تطبيق تصميم مميز ومنظم هندسياً

#### 2.3 إدارة أقسام الهوية المهنية (من لوحة الأدمن)
- [ ] إضافة أزرار تحكم كاملة (إضافة/حذف/تعديل) للأقسام التالية:
  - [ ] قسم مسيرتي المهنية
  - [ ] قسم من أنا (مع تعديل النص والصورة الشخصية)
  - [ ] قسم المؤهلات والشهادات
  - [ ] قسم الخدمات
  - [ ] قسم تواصل معي
  - [ ] التحكم الكامل بجميع الروابط المدمجة

#### 2.4 إدارة اللوجو الرسمي
- [ ] تغيير اللوجو من لوحة التحكم
- [ ] تحديث تلقائي في المنصة بالكامل

#### 2.5 إدارة معرض الصور والشركاء (Dynamic Sliders)
- [ ] لوحة تحكم لقسم "لحظات شكلت المسيرة"
  - [ ] إضافة وحذف الصور
  - [ ] ضبط أبعادها وتنسيقها برمجياً
- [ ] لوحة تحكم لقسم "شركاء صنعت معهم الأثر"
  - [ ] إضافة وحذف لوجوهات الشركات
  - [ ] تنسيق عرض الشركاء

#### 2.6 تحديث قسم الإحصائيات
- [ ] ربط الأرقام الخاصة بـ:
  - [ ] عدد القطاعات
  - [ ] عدد الدول
  - [ ] عدد سنوات الخبرة
- [ ] تعديل الأرقام من لوحة التحكم في أي وقت

#### 2.7 إضافات قسم "من أنا"
- [ ] إضافة زر لتحميل الملف الشخصي / السيرة الذاتية (CV)
- [ ] إمكانية رفع وتحديث الملف من الأدمن
- [ ] إضافة خيار لرفع لوجوهات الاعتمادات الدولية (مثل PMP)
- [ ] لوجوهات الجهات المانحة للكورسات والدورات
- [ ] عرض الاعتمادات بوضوح وبشكل منظم

### 🟠 المرحلة 3️⃣: منظومة التسجيل وتفعيل الحسابات (Authentication & Onboarding)

**الأولوية:** عالية | **المدة المتوقعة:** 2-3 أسابيع

#### 3.1 تغيير طريقة دخول السوبر أدمن
- [ ] تعديل حقل تسجيل الدخول للسوبر أدمن
- [ ] استخدام اسم المستخدم (Username) بدلاً من البريد الإلكتروني
- [ ] مثال: `eslamselmi` بدلاً من `email@example.com`

#### 3.2 نظام تفعيل الحسابات عبر الواتس آب
- [ ] إيقاف إرسال أي إيميلات تفعيل للمتدربين تماماً
- [ ] عند إنشاء حساب، عرض نافذة منبثقة (Popup) احترافية
- [ ] النافذة تخبر المتدرب أن التفعيل يتم عبر الواتس آب
- [ ] عرض زر واضح: "اطلب تفعيل حسابك الآن"
- [ ] الضغط عليه يحول المستخدم مباشرة إلى واتس آب الإدارة
- [ ] رسالة مجهزة مسبقاً
- [ ] إمكانية تغيير رقم الواتس آب من لوحة التحكم
- [ ] التحديث التلقائي في جميع الأماكن المرتبطة برقم الواتس

#### 3.3 نظام التنبيهات الذكي للأدمن والمتدرب (Glow & Audio Effects)
- [ ] إضافة نظام تنبيهات مرئي (Glow Effect / Badges)
- [ ] توهج على أقسام لوحة الأدمن عند وجود طلبات جديدة أو تعديلات
- [ ] تنبيهات صوتية خفيفة وراقية (Audio Chime)
- [ ] تفعيل للأدمن والمتدرب عند وصول التنبيهات الهامة والحرجة فقط
- [ ] كود برمجي أصلي وخفيف جداً
- [ ] عدم استهلاك التوكنز أو التأثير على السرعة

#### 3.4 إضافة نافذة ترحيبية / إعلانية عامة (Global Popup)
- [ ] خاصية من الأدمن لإنشاء نافذة Popup تظهر للزوار
- [ ] تحكم كامل بـ:
  - [ ] شكل النافذة
  - [ ] تفعيلها وأرشفتها
  - [ ] وقت الظهور والاختفاء
- [ ] زر إغلاق (X) واضح للمستخدم

### 🟡 المرحلة 4️⃣: نظام إدارة الكورسات والطلاب (Courses & Students Management)

**الأولوية:** عالية | **المدة المتوقعة:** 3-4 أسابيع

#### 4.1 إعادة هيكلة صفحات الكورسات بالأدمن
**تقسيم إدارة الكورس إلى جزأين منفصلين:**

**أ) جزء الإعدادات (Settings)**
- [ ] رفع المحتوى
- [ ] التعديلات الفنية للكورس

**ب) مركز المتابعة والعمليات (Operations Dashboard)**
- [ ] استعراض المتدربين المسجلين
- [ ] إدارة المديونيات
- [ ] إدارة التكليفات ومشاريع التخرج

#### 4.2 خيارات الأرشفة والحذف الذكي للكورسات
- [ ] **أوبشن الحذف الكامل:**
  - [ ] حذف الكورس نهائياً من المنصة
  - [ ] طلب من الأدمن كتابة كلمة المرور لتأكيد الحذف (أمان إضافي)

- [ ] **أوبشن الأرشفة:**
  - [ ] إخفاء الكورس من صفحات المتدربين
  - [ ] منع الاشتراكات الجديدة
  - [ ] الحفاظ على الكورس للمتدربين المشتركين سابقاً

#### 4.3 تحديث واجهة الكورسات للمتدرب
- [ ] إعادة تصميم قسم "الكورسات المتاحة"
- [ ] تصميم مميز وجذاب
- [ ] حذف تبويب "أحدث الإضافات" نهائياً
- [ ] معالجة عدم ظهوره على الموبايل

#### 4.4 إعدادات الكورس المتقدمة (من الأدمن)
- [ ] إضافة حقول لكتابة الأهداف (Objectives) على هيئة نقاط
- [ ] تحديد الفئة المستهدفة (Target Audience) لكل كورس
- [ ] تصميم مميز للحقول الجديدة
- [ ] خيار "تصفح محتويات الكورس" للمتدرب قبل الانضمام

#### 4.5 إصلاحات تقنية وروابط الكورسات
- [ ] تغيير جملة التحذير إلى صياغة تمنح ثقة
  - [ ] من: "هذا المحتوى يفتح في صفحة جديدة للحفاظ على أمان تصفحك..."
  - [ ] إلى: "يتم الآن فتح الرابط في علامة تبويب جديدة لتصفح مرن ومستمر"
- [ ] إصلاح مشكلة الروابط الخارجية (Redirect Error)
  - [ ] فتح الرابط الخارجي مباشرة (مثل Google)
  - [ ] دون دمج رابط المنصة قبله بشكل خاطئ

### 🟡 المرحلة 5️⃣: نظام التكليفات ومشاريع التخرج (Assignments & Capstone Projects)

**الأولوية:** متوسطة عالية | **المدة المتوقعة:** 2-3 أسابيع

#### 5.1 نقل ونظامة التقييم
- [ ] نقل استعراض وتقييم التكليفات ومشاريع التخرج
- [ ] من: إعدادات الكورس
- [ ] إلى: ملف إدارة الطالب / الملف الشخصي
- [ ] بمجرد رفع المتدرب للتكليف:
  - [ ] يستطيع الأدمن الدخول على ملف الطالب
  - [ ] استعراض الملفات
  - [ ] تقييمها من هناك مباشرة

#### 5.2 تبويب الفرز الذكي (داخل الكورس)
- [ ] إنشاء تبويب خاص بالتكليفات والمشاريع
- [ ] تقسيم إلى جزأين:
  - [ ] قسم المشاريع منفصل
  - [ ] قسم التكليفات منفصل
- [ ] ترتيب برمجي:
  - [ ] البدء دائماً بـ "المشاريع التي لم تقيم بعد"
  - [ ] ضمان عدم نسيان أي طالب

#### 5.3 لوحة المتابعة الشاملة (Tracking Dashboard)
- [ ] إنشاء تبويب (Matrix) يوضح للأدمن:
  - [ ] من من الطلاب لم يسلم التكليفات أو المشاريع
  - [ ] نوع التقصير (تكليف رقم X أو مشروع التخرج)
  - [ ] حالة التقييم والتسليم

#### 5.4 نظام تنبيهات ذكي للمتدرب (Browser Notifications)
- [ ] تنبيهات على المتصفح عند:
  - [ ] اقتراب موعد تسليم التكليف
  - [ ] نفس يوم التسليم
  - [ ] تجاوز موعد التسليم
- [ ] التنبيه لا يختفي إلا بعد رفع التكليف بنجاح
- [ ] تنبيهات واضحة وجذابة

#### 5.5 تحسين وضوح التنبيهات
- [ ] تحسين وضوح ولون نص التنبيه
- [ ] خاص بـ: (ضبط إعدادات مشاركة الرابط ليكون عام)
- [ ] جعل النص بارزاً للمتدرب قبل إرفاق الرابط

### 🟡 المرحلة 6️⃣: النظام المالي والتقسيط والأدوات التسويقية (Financials & Marketing Tools)

**الأولوية:** متوسطة | **المدة المتوقعة:** 2-3 أسابيع

#### 6.1 نظام إشعارات التقسيط
- [ ] إذا تم ضبط الكورس على أقساط (مثال: قسطين)
- [ ] يظهر للمتدرب تنبيه/ملحوظة واضحة بجانب خيار التقديم
- [ ] توضيح:
  - [ ] عدد الأقساط
  - [ ] قيمة كل قسط بناءً على إعدادات الكورس

#### 6.2 لوحة تتبع الأقساط
- [ ] عند دفع المتدرب للقسط الأول (مثال: 500)
- [ ] واعتماده من الأدمن:
  - [ ] يظهر في حساب المتدرب: "القسط الأول: مدفوع ومُعتمد"
  - [ ] يظهر في لوحة الأدمن: نفس المعلومة
  - [ ] يظل "القسط الثاني: مستحق" واضحاً

#### 6.3 لوحة المديونيات الذكية
- [ ] إمكانية للأدمن معرفة المتعثرين ماليّاً بناءً على كل كورس
- [ ] زر "تذكير بالسداد"
- [ ] تحديد توقيت وجدولة وصول التذكير تلقائياً

#### 6.4 أداة المراسلة الجماعية الذكية (Bulk Messaging & Templates)
- [ ] إضافة ميزة بالسوبر أدمن:
  - [ ] إرسال رسالة واتس آب مجمعة
  - [ ] إرسال إيميل مجمع
  - [ ] لكافة المسجلين أو المهتمين بكورس معين
  - [ ] عبر نافذة Popup مخصصة

#### 6.5 نظام القوالب الجاهزة (Templates)
- [ ] دعم الأداة بقوالب رسائل معدّة مسبقاً:
  - [ ] رسائل تذكير
  - [ ] عروض
  - [ ] إعلانات
- [ ] سهولة الاختيار والتعديل السريع

#### 6.6 التحكم في الجلسات المجانية
- [ ] خيار (إظهار / إخفاء) لكل ما يخص "جلسات مجانية معي"
- [ ] من لوحة التحكم بضغطة زر واحدة

### 🟡 المرحلة 7️⃣: نظام الشهادات والتحقق (Certification & Verification System)

**الأولوية:** متوسطة | **المدة المتوقعة:** 3-4 أسابيع

#### 7.1 الرفع اليدوي للشهادات
- [ ] عند رفع شهادة للمتدرب يدوياً
- [ ] تحديد لغة الشهادة:
  - [ ] عربي فقط
  - [ ] إنجليزي فقط
  - [ ] الشهادتين معاً
- [ ] الظهور في حساب المتدرب بناءً على التحديد

#### 7.2 نظام إصدار الشهادات التلقائي الذكي (Dynamic PDF Generator)
- [ ] إلغاء المظهر القديم
- [ ] تطوير محرك الشهادات
- [ ] إمكانية رفع "تصميم شهادة فارغ" (خلفية)
  - [ ] عربي وإنجليزي
  - [ ] لكل كورس من الأدمن
- [ ] تحديد متغيرات سحرية (Dynamic Tags):
  - [ ] [اسم المتدرب بالعربي]
  - [ ] [اسم المتدرب بالإنجليزي]
  - [ ] [تاريخ الاصدار]
  - [ ] [اسم الكورس]
  - [ ] (يمكن إضافة متغيرات أخرى حسب الحاجة)
- [ ] تركيب البيانات بذكاء فوق التصميم الفارغ
- [ ] استخدام خط احترافي

#### 7.3 دعم LinkedIn Share
- [ ] نص جاهز (عربي وإنجليزي) في إعدادات الكورس
- [ ] يظهر للمتدرب عند حصوله على الشهادة
- [ ] المتدرب ينسخ ويشاركه مباشرة على LinkedIn
- [ ] دمج اسمه تلقائياً بالرسالة

#### 7.4 نظام التحقق من الشهادات (Certificate Verification)
- [ ] إضافة زر احترافي أسفل الشهادة
- [ ] الزر باسم: "التحقق من صحة الشهادة"
- [ ] عند الضغط عليه:
  - [ ] حقل لإدخال الرقم التسلسلي (Serial Number)
- [ ] إذا كانت الشهادة صحيحة:
  - [ ] Popup بتصميم مبهر
  - [ ] علامة توثيق (Verified Badge)
  - [ ] اسم المتدرب
  - [ ] تاريخ الحصول عليها
  - [ ] تأكيد مصداقية المنصة

- [ ] إذا تم إنشاء حساب لمدرب وتعيينه لكورس معين:

- [ ] حجب أي أقسام أو بيانات أخرى خارج نطاق صلاحياته نهائياً
- [ ] عدم ظهور هذه البيانات في شاشة المدرب
